1: Управление заказами на фабрике
Создайте систему для управления заказами на фабрике. Фабрика производит разные товары, и каждый заказ имеет свой тип продукции.

Создайте базовый класс Order, который содержит:
Номер заказа,
Имя клиента,
Список товаров в заказе.

Создайте производные классы:
ElectronicsOrder: заказ на электронику. Добавьте поле для указания типа устройства (например, телефон, телевизор).
FurnitureOrder: заказ на мебель. Добавьте поле для указания типа мебели (стол, диван, и т.д.).
ClothingOrder: заказ на одежду. Добавьте поле для размера одежды.
Задачи:

Каждый класс заказа должен содержать метод для расчёта итоговой стоимости заказа (например, на основе стоимости товаров и количества единиц товара).
Напишите метод для вывода информации о каждом заказе, включая общую стоимость.
Добавьте возможность вывода всех заказов в одном списке с использованием полиморфизма, чтобы все типы заказов можно было хранить в массиве или списке типа Order[].


2:Система персонажей в игре
Разработайте систему персонажей для игры, каждый из которых имеет свои уникальные характеристики и способности.

Создайте базовый класс Character, который содержит общие параметры:
Имя персонажа,
Уровень здоровья,
Сила атаки.

Создайте производные классы:
Warrior: воин, который может носить щит и имеет дополнительное поле для защиты.
Mage: маг, который может использовать магическую атаку и имеет поле для количества маны.
Archer: лучник, который может стрелять из лука и имеет поле для количества стрел.
Задачи:

Напишите метод атаки, который зависит от типа персонажа (например, лучник может стрелять на дальнее расстояние, а воин наносит мощный удар вблизи).
Реализуйте метод для вывода текущего состояния персонажа (здоровье, атака, и т.д.).
Модифицируйте систему так, чтобы персонажи могли сражаться между собой, и их здоровье уменьшалось в зависимости от нанесённого урона.
